## How to Contribute

We welcome all kinds of contributions ‚Äî code, content, bug reports, design, and documentation.

### 1. Fork & Clone

```bash
git clone https://github.com/xenohuru/xenohuru-web.git
cd twenzetu-safari-web
```

### 2. Create a Feature Branch

Use a descriptive branch name:

```bash
git checkout -b feature/add-zanzibar-images
git checkout -b fix/weather-widget-mobile
git checkout -b docs/update-api-readme
```

### 3. Make Your Changes

- Keep changes focused and minimal
- Follow the existing code style (vanilla JS, Tailwind CSS v4, semantic HTML)
- Test on both desktop and mobile before submitting

### 4. Commit with Conventional Commits

```bash
git commit -m "feat: add gallery for Serengeti attraction"
git commit -m "fix: correct GPS coordinates for Ngorongoro Crater"
git commit -m "docs: improve quick start instructions"
```

| Prefix | Use for |
|--------|---------|
| `feat:` | New feature or content |
| `fix:` | Bug fix |
| `docs:` | Documentation only |
| `style:` | Formatting, no logic change |
| `refactor:` | Code restructure, no behaviour change |
| `chore:` | Build tasks, dependencies |

### 5. Open a Pull Request

Push your branch and open a PR against `main`. Describe:
- **What** you changed
- **Why** the change is needed
- Screenshots or recordings for UI changes

---

## Reporting Bugs

Open an issue on [GitHub Issues](https://github.com/xenohuru/xenohuru-web/issues) and include:

- Steps to reproduce
- Expected vs actual behaviour
- Browser/device info
- Screenshots if relevant

---

## Suggesting Features

Open a [GitHub Discussion](https://github.com/xenohuru/xenohuru-web/discussions) or issue labelled `enhancement`. Describe the use case and why it benefits Tanzania tourism visitors.

---

## Contribution Checklist

Before submitting a PR, make sure:

- [ ] Code follows the existing style
- [ ] No console errors on desktop and mobile
- [ ] New HTML pages include proper `<meta>` tags and breadcrumb
- [ ] Images use `loading="lazy"` and have descriptive `alt` text
- [ ] No hardcoded secrets or API keys committed
- [ ] PR description clearly explains the change
